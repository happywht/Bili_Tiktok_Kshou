# 错误排查指南

## 🔴 问题：API 连接失败 (ERR_CONNECTION_REFUSED)

### 症状
```
API Error: Network Error
Failed to load resource: net::ERR_CONNECTION_REFUSED
```

### 原因
前端无法连接到后端API服务器（端口8000）。

### 解决方案

#### 1. 确认后端服务是否启动

**方法1：查看API文档**
```
浏览器访问: http://localhost:8000/docs
```
- ✅ 能访问：后端正常
- ❌ 无法访问：后端未启动

**方法2：检查健康检查接口**
```bash
curl http://localhost:8000/api/v1/health
```

#### 2. 启动后端服务

**Windows：**
```bash
# 方法1：使用启动脚本（推荐）
start.bat

# 方法2：手动启动
python -m uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

**Linux/Mac：**
```bash
# 方法1：使用启动脚本（推荐）
./start.sh

# 方法2：手动启动
python3 -m uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

#### 3. 确认端口占用

**Windows：**
```powershell
# 查看端口8000占用情况
netstat -ano | findstr :8000

# 终止进程（替换PID为实际进程ID）
taskkill /PID <PID> /F
```

**Linux/Mac：**
```bash
# 查看端口8000占用情况
lsof -i:8000

# 终止进程（替换PID为实际进程ID）
kill -9 <PID>
```

#### 4. 检查防火墙设置

确保防火墙允许以下端口：
- `8000` - 后端API
- `5173` - 前端开发服务器

## 🟠 问题：API 返回错误

### 检查后端日志

启动后端的控制台窗口会显示详细的错误信息，包括：
- 请求路径
- 错误类型
- 堆栈跟踪

### 常见错误及解决方案

| 错误类型 | 说明 | 解决方案 |
|---------|------|---------|
| `CookieExpiredError` | B站登录已过期 | 更新 `.env` 中的 `BILIBILI_SESSDATA` |
| `RateLimitError` | 请求过于频繁 | 降低请求频率，稍后重试 |
| `ContentNotFoundError` | 视频不存在 | 检查视频ID是否正确 |
| `InvalidParameterError` | 参数无效 | 检查请求参数格式 |

## 💡 优化建议

### 开发环境

1. **分离启动后端和前端**
   - 后端：独立窗口运行 `python -m uvicorn api.main:app --reload`
   - 前端：独立窗口运行 `cd web && npm run dev`

2. **使用环境变量**
   创建 `.env` 文件在项目根目录：
   ```env
   VITE_API_URL=http://localhost:8000/api/v1
   ```

### 生产环境

1. **构建前端**
   ```bash
   cd web
   npm run build
   ```

2. **部署后端**
   ```bash
   # 使用 Gunicorn
   pip install gunicorn
   gunicorn api.main:app -w 4 -b 0.0.0.0:8000
   ```

3. **使用 Nginx 反向代理**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       # 前端静态文件
       location / {
           root /path/to/web/dist;
           try_files $uri $uri/ /index.html;
       }
       
       # 后端API
       location /api/ {
           proxy_pass http://localhost:8000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
       }
   }
   ```
