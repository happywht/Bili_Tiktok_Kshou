# 删除抖音和小红书模块说明

## 修改日期
2026-03-31

## 修改内容

### 1. 后端修改

#### `api/routers/search.py`
- 删除了抖音和小红书相关的异常导入（`DouyinCookieExpiredError`, `XiaohongshuCookieExpiredError`, `DouyinNotAvailableError`, `XiaohongshuNotAvailableError`, `CaptchaDetectedError`）
- 删除了抖音和小红书的平台图标配置
- 修改了平台验证逻辑，只保留 `bilibili`
- 删除了抖音和小红书专用的搜索路由（`/search/douyin` 和 `/search/xiaohongshu`）
- 删除了图片代理中的抖音和小红书域名白名单
- 简化了搜索逻辑，移除了平台特定的条件判断

#### `api/services/factory.py`
- 删除了 `DouyinService` 和 `XiaohongshuService` 的导入
- 从服务注册表中移除了抖音和小红书
- 移除了平台映射中的抖音和小红书选项
- 简化了服务创建逻辑
- 删除了抖音和小红书的便捷函数
- 简化了平台名称映射

#### `api/config.py`
- 将应用名称从 "Multi-Platform Video Search API" 改为 "Bilibili Video Search API"
- 删除了抖音相关配置（`DOUYIN_COOKIES`, `DOUYIN_TTWID`, `DOUYIN_HEADLESS`）
- 删除了小红书相关配置（`XIAOHONGSHU_COOKIES`, `XIAOHONGSHU_HEADLESS`）
- 删除了 MediaCrawler 相关配置（`MEDIACRAWLER_BRIDGE_PATH`, `MEDIACRAWLER_TIMEOUT`, `MEDIACRAWLER_HEADLESS`）
- 简化了 `get_platform_config` 方法，只保留 B站配置

#### `api/services/summary_service.py`
- 更新了文档字符串，说明只支持B站
- 删除了抖音和小红书的 URL 模式
- 更新了 `summarize_video` 函数的错误提示

#### `api/exceptions.py`
- 删除了 `DouyinCookieExpiredError` 异常类
- 删除了 `XiaohongshuCookieExpiredError` 异常类
- 删除了 `DouyinNotAvailableError` 异常类
- 删除了 `XiaohongshuNotAvailableError` 异常类
- 删除了 `CaptchaDetectedError` 异常类
- 简化了 `create_platform_error` 函数，移除抖音和小红书的错误码映射
- 简化了 `detect_error_from_response` 函数

#### `api/services/base.py`
- 从 `PlatformType` 枚举中删除了 `DOUYIN` 和 `XIAOHONGSHU`

#### 删除的文件

## 注意事项

1. **环境变量**：`.env` 文件中的 `LLM_API_KEY` 仍然需要配置才能使用视频总结功能
2. **B站Cookie**：`BILIBILI_SESSDATA` 是可选的，但配置后可以访问更多内容
3. **yt-dlp依赖**：视频总结功能需要 yt-dlp，启动脚本会自动检查并安装
4. **MediaCrawler目录**：可以手动删除 `tools/MediaCrawler/` 目录及其内容（1950+文件）
