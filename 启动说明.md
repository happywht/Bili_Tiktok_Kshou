# 启动说明（Windows中文路径）

## 🔧 编码问题修复

已修复 `api/config.py` 中的中文路径编码问题，现在支持 Windows 中文路径。

## 🚀 启动方法

### 方法1：使用 start.bat（推荐）

直接双击运行项目根目录下的 `start.bat` 文件。

### 方法2：手动启动后端

在项目根目录打开命令提示符（cmd），执行：

```cmd
set PYTHONIOENCODING=utf-8
set PYTHONPATH=%~dp0
python -m uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

### 方法3：使用 PowerShell

在项目根目录打开 PowerShell，执行：

```powershell
$env:PYTHONIOENCODING = "utf-8"
$env:PYTHONPATH = Get-Location
python -m uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

## 🌐 访问地址

启动成功后访问：
- 前端：http://localhost:5173
- 后端API文档：http://localhost:8000/docs

## 📝 关于"B站未准备好"和"AI总结未准备好"的提示

这是启动脚本中的模块检查，属于**正常提示**，不影响使用：

- **B站未准备好**：可能只是导入检查失败，但实际功能正常
- **AI总结未准备好**：可能是 yt-dlp 或 API Key 配置检查

## ⚠️ 常见问题

### 问题1：启动失败，提示编码错误

**解决**：使用命令行手动启动，确保设置了环境变量：
```cmd
set PYTHONIOENCODING=utf-8
```

### 问题2：模块导入失败

**解决**：检查是否正确设置了 PYTHONPATH
```cmd
set PYTHONPATH=D:\工作\城建院\2603\B站信息检索\bilibili_search
```

### 问题3：前端无法连接后端

**解决**：
1. 确认后端已启动（访问 http://localhost:8000/docs）
2. 检查端口是否被占用（8000）
3. 查看 `docs/错误排查指南.md`
